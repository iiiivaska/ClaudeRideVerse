# Прочие пакеты
> WorkoutKit, TrackExport, PersistenceCore, DesignSystem. Вспомогательные переиспользуемые пакеты, не входящие в основные группы.

---

## WorkoutKit

**Назначение:** обёртка над HealthKit -- `HKWorkoutSession` wrapper, route insertion fix.

| Свойство | Значение |
|---|---|
| Зависимости | HealthKit |
| Публичный контракт | HKWorkoutSession wrapper, route insertion fix |

Инкапсулирует работу с HealthKit для записи тренировок. Включает workaround для критического gotcha FB18603581 (route insertion). Отвечает за:
- Запуск/остановку `HKWorkoutSession`.
- Вставку маршрута (`HKWorkoutRouteBuilder`).
- Сохранение тренировки в HealthKit.

[[LocationKit|LocationRecording]] не вставляет в `HKWorkoutRouteBuilder` -- это обязанность WorkoutKit.

---

## TrackExport

**Назначение:** GPX reader/writer, FIT writer. Импорт и экспорт треков.

| Свойство | Значение |
|---|---|
| Зависимости | -- (без внешних зависимостей) |
| Публичный контракт | GPX reader/writer, FIT writer |

Пакет без внешних зависимостей. Поддерживает:
- **GPX** -- чтение и запись (для импорта/экспорта треков).
- **FIT** -- запись (для совместимости с Garmin Connect, Strava и другими платформами).

## PersistenceCore

**Назначение:** GRDB helpers, миграции, WAL-конфигурация, observation helpers.

| Свойство | Значение |
|---|---|
| Зависимости | GRDB |
| Публичный контракт | WAL, миграции, observation helpers |

Общие утилиты для работы с GRDB, не привязанные к доменной модели FogRide:
- Конфигурация WAL-режима (Write-Ahead Logging).
- Базовая инфраструктура миграций.
- Helpers для GRDB observation (async stream обёртки).

Используется в [[Domain|FogRidePersistence]] для конкретных таблиц и репозиториев.

---

## DesignSystem

**Назначение:** FogRide design tokens -- цвета, типографика, общие компоненты.

| Свойство | Значение |
|---|---|
| Зависимости | SwiftUI |
| Публичный контракт | FogRide tokens: colors, typography, components |

Содержит:
- Цветовые токены (Liquid Glass palette).
- Типографику (стили текста).
- Общие UI-компоненты, используемые в нескольких feature-модулях.

Если `TripDetailFeature` нужно поделиться кнопкой с `RecordingFeature`, кнопка выносится сюда (см. [[Антипаттерны]]).
