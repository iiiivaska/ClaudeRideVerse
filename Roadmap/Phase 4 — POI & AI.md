# Phase 4 — POI & AI
> 8-10 недель (месяцы 13-14). ML-рекомендации и POI: "Спланируй маршрут, который откроет максимум нового тумана."

## Цель

Добавить слой точек интереса (POI) и ML-рекомендации маршрутов. Killer feature фазы — интеллектуальный планировщик маршрутов, который предлагает путь с максимальным количеством новых hex-ячеек для открытия. Это превращает fog of war из пассивной визуализации в активный геймплей.

## Go/No-Go критерии

- Успешный релиз [[Phase 3 — Backend]]
- Бэкенд (Supabase + PostGIS) стабилен и масштабируется
- Достаточно данных о поездках для обучения рекомендательных моделей

## Задачи (Jira)

Задачи в Jira пока не декомпозированы — эпик SCRUM-10 содержит только описание. Предполагаемые направления:

- POI-слой на карте (интересные места, веломагазины, водные точки)
- ML-модель для предсказания "интересности" маршрута
- **Планировщик маршрутов "Maximum Fog Unlock"** — killer feature
- Персональные рекомендации на основе истории поездок
- Интеграция POI с heatmap данными из [[Phase 3 — Backend]]

## Ключевые решения

- Детальная декомпозиция будет выполнена ближе к началу фазы (через ~12 месяцев)
- ML-инференс предпочтительно на устройстве (Core ML) для приватности и офлайн-работы
- POI-данные могут комбинировать открытые источники (OSM) и пользовательский контент
- Планировщик маршрутов использует [[HexKit]] для расчёта покрытия и оптимизации пути
